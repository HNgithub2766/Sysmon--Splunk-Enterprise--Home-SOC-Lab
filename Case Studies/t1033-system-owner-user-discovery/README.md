## Objective

Simulate attacker reconnaissance activity by identifying the current user and logged-on sessions on a Windows host.

## MITRE ATT&CK Mapping

* Technique: T1033 - System Owner/User Discovery
* Tactic: Discovery

## Lab Environment

* Windows 11 VM
* Sysmon
* Splunk Enterprise
* Atomic Red Team

## Atomic Test Executed

```powershell
Invoke-AtomicTest T1033
```

## Detection Logic

### Splunk Search

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=1
(Image="*\\whoami.exe" OR CommandLine="*whoami*")
```

### Alert Purpose

Detect execution of user discovery commands commonly used by attackers during the reconnaissance phase of an intrusion.

## Investigation Steps

1. Reviewed Sysmon Event ID 1 process creation logs.
2. Identified execution of whoami.exe.
3. Reviewed command line arguments.
4. Verified the executing user account.
5. Correlated activity with Atomic Red Team test execution.

## Findings

* whoami.exe executed successfully.
* User account identified as DESKTOP-QSSM57Q\Hafsa.
* Activity originated from the local host.
* No additional malicious behaviour observed.

## Analysis

The activity matches MITRE ATT&CK T1033 (System Owner/User Discovery). This technique is commonly used by threat actors after gaining initial access to determine user context and privilege levels before moving laterally or escalating privileges.

## Response Actions

* Investigated process execution.
* Verified activity was generated by authorised testing.
* No containment required.

## Conclusion

This detection successfully identified user discovery activity generated by Atomic Red Team. The detection can be used as a low-severity reconnaissance alert and may be enhanced by correlating with other discovery techniques such as T1082 and T1016.
